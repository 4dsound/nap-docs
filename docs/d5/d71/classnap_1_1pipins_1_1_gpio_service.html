Description

GpioService starts a new thread from which calls to WiringPI API methods can be made via the GpioPin resource. Write calls will always be asynchronous and not block the calling thread. Read calls are synchronous and perform a mutex lock to ensure thread-safety. The GpioService uses "wiringPiSetupGpio()" which means the service uses the Broadcom GPIO pin numbers directly with no re-mapping. See https://pinout.xyz/ to help with mapping. The GpioService wraps the core functionality of wiringPi, see : http://wiringpi.com/reference/
